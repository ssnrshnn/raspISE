# RaspISE

A Cisco ISE–inspired Network Access Control (NAC) / AAA server designed to run on a **Raspberry Pi Zero 2W** with a 2.4" ILI9341 TFT display (240×320 px).

> **Home-lab / learning project.** This is not production-grade security infrastructure.

---

## Feature Overview

| Feature | Details |
|---|---|
| **RADIUS** | PAP, CHAP, MAB; VLAN assignment (RFC 3580); Accounting |
| **EAP-PEAP / EAP-TLS** | Via optional FreeRADIUS integration |
| **TACACS+** | Full RFC 8907 — Authentication, Authorization, Accounting |
| **Guest Portal** | Captive portal with registration, timed sessions, Wi-Fi QR code |
| **Device Profiler** | Passive DHCP/ARP sniffing, OUI vendor lookup, OS fingerprinting |
| **Policy Engine** | Condition-based rules (username, group, MAC, time, device type, NAS IP) |
| **Live TFT Dashboard** | 5 rotating screens — auth log, system stats, sessions, network, QR code |
| **Web Admin UI** | Bootstrap 5 dark UI at `:8080` (Jinja2 server-side rendering) |
| **REST API** | Full CRUD with JWT auth at `:8081` |

## Hardware Requirements

| Component | Spec |
|---|---|
| SBC | Raspberry Pi Zero 2W |
| RAM | 512 MB (onboard) |
| Storage | 8 GB+ microSD (Class 10 / A1) |
| Display | 2.4" TFT SPI — ILI9341 or ST7789 controller, 240×320 px |
| Power | 5V / 2A USB-C |
| Network | Built-in Wi-Fi (wlan0) or USB Ethernet for NAS |

### TFT Wiring (Pi Zero 2W — 40-pin header)

```
TFT Pin   Pi Pin   BCM/Function
──────────────────────────────────
VCC    →  Pin 1    3.3V
GND    →  Pin 6    GND
CLK    →  Pin 23   BCM11 / SPI0_SCLK
MOSI   →  Pin 19   BCM10 / SPI0_MOSI
CS     →  Pin 24   BCM8  / SPI0_CE0
DC     →  Pin 18   BCM24
RST    →  Pin 22   BCM25
LED    →  Pin 17   3.3V  (or PWM for dimming)
```

## Quick Start

### 1. Flash & boot Raspberry Pi OS Bookworm 64-bit Lite

Use [Raspberry Pi Imager](https://www.raspberrypi.com/software/) and pre-configure Wi-Fi + SSH.

### 2. Clone the repository

```bash
git clone https://github.com/you/raspise.git
cd raspise
```

### 3. Run the installer

```bash
sudo bash scripts/install.sh
```

This will:
- Install system packages and enable SPI
- Create `/opt/raspise/` with a Python venv
- Write default config to `/etc/raspise/config.yaml`
- Download the IEEE OUI database
- Install and enable `raspise.service` + `raspise-display.service`

### 4. Edit configuration

```bash
sudo nano /etc/raspise/config.yaml
```

Minimum required edits:
- `radius.clients` — add your NAS/AP IP and shared secret
- `tacacs.clients` — add your router/switch IP and key
- `display.driver` — set to `ili9341`, `st7789`, or `simulation`
- `web.secret_key` — auto-generated by installer (leave as-is)

### 5. Optional: Set up the display

```bash
# ILI9341 (default) or st7789
sudo bash scripts/setup_display.sh ili9341
sudo reboot
```

### 6. Optional: EAP-PEAP / EAP-TLS via FreeRADIUS

```bash
sudo bash scripts/setup_freeradius.sh
sudo systemctl restart raspise
```

### 7. Start RaspISE

```bash
sudo systemctl start raspise
sudo systemctl start raspise-display   # if display is connected
```

### 8. Open the admin UI

```
http://<pi-ip>:8080
Username: admin
Password: RaspISE@admin1   ← change this immediately!
```

## Authentication Methods

| Method | Protocol | Notes |
|---|---|---|
| PAP | RADIUS | Plaintext password in RADIUS packet (encrypted by shared secret) |
| CHAP | RADIUS | Challenge-response; requires cleartext password storage |
| MAB | RADIUS | MAC address as username — for printers, IoT, etc. |
| Web Auth | HTTP | Guest captive portal with timed sessions |
| EAP-PEAP/MSCHAPv2 | RADIUS | **Requires FreeRADIUS** (`setup_freeradius.sh`) |
| EAP-TLS | RADIUS | **Requires FreeRADIUS** — certificate-based 802.1X |
| PAP/ASCII | TACACS+ | For router/switch CLI authentication |

## Policy Engine

Policies are evaluated in priority order. The first matching policy wins. Default action is **DENY**.

### Condition Types

| Type | Example value | Operators |
|---|---|---|
| `username` | `"alice"` | `equals`, `startswith`, `contains`, `regex` |
| `group` | `"network-admins"` | `equals`, `in` |
| `mac` | `"aa:bb:cc:dd:ee:ff"` | `equals`, `startswith` |
| `device_type` | `"iPhone"` | `equals`, `contains` |
| `nas_ip` | `"192.168.1.1"` | `equals`, `startswith` |
| `time` | `"08:00-18:00"` | `between` |
| `always` | `true` | *(catch-all)* |

### Example Policy (JSON)

```json
[
  {
    "type": "group",
    "operator": "equals",
    "value": "employees"
  },
  {
    "type": "time",
    "operator": "between",
    "value": "07:00-22:00"
  }
]
```

This policy permits members of **employees** group during business hours and assigns them to VLAN 10.

## TFT Display Screens

The display cycles through 5 screens every 8 seconds (configurable):

| Screen | Content |
|---|---|
| **Auth Log** | Last 8 authentication events with ACCEPT/REJECT status |
| **System Stats** | CPU %, RAM %, Disk usage bars, temperature, uptime |
| **Sessions** | Active session count, today's accept/reject/guest totals |
| **Network** | All network interfaces with IP addresses |
| **QR Code** | Guest Wi-Fi QR code (WPA-format, scannable) |

Set `display.driver: simulation` to write frames as PNG files to `/tmp/raspise_display/` for development without hardware.

## REST API Reference

Base URL: `http://<pi-ip>:8081`

All endpoints except `/auth/login` require `Authorization: Bearer <token>`.

### Authentication
| Method | Path | Description |
|---|---|---|
| POST | `/auth/login` | Get JWT token (`{"username":"...","password":"..."}`) |

### Users
| Method | Path | Description |
|---|---|---|
| GET | `/users` | List all users |
| POST | `/users` | Create user |
| GET | `/users/{id}` | Get user |
| PUT | `/users/{id}` | Update user |
| DELETE | `/users/{id}` | Delete user |

### Devices
| Method | Path | Description |
|---|---|---|
| GET | `/devices` | List profiled devices |
| GET | `/devices/{id}` | Get device |
| PUT | `/devices/{id}` | Update (authorize/deny) |

### Policies
| Method | Path | Description |
|---|---|---|
| GET | `/policies` | List policies |
| POST | `/policies` | Create policy |
| PUT | `/policies/{id}` | Update policy |
| DELETE | `/policies/{id}` | Delete policy |

### Logs & Sessions
| Method | Path | Description |
|---|---|---|
| GET | `/logs/auth` | Auth log (paginated) |
| GET | `/logs/tacacs` | TACACS+ log (paginated) |
| GET | `/sessions` | Active RADIUS sessions |
| DELETE | `/sessions/{id}` | Force-terminate session |
| GET | `/dashboard` | Summary stats |

## Configuration Reference

`/etc/raspise/config.yaml`:

```yaml
server:
  host: "0.0.0.0"
  log_level: "INFO"       # DEBUG | INFO | WARNING | ERROR

database:
  path: "/var/lib/raspise/raspise.db"

radius:
  enabled: true            # set false if using FreeRADIUS
  port: 1812
  clients:
    - name: my-ap
      ip: 192.168.1.1
      secret: "changeme"
  vlans:
    employees: 10
    guests: 20
    iot: 30

tacacs:
  enabled: true
  port: 49
  clients:
    - name: core-switch
      ip: 192.168.1.254
      key: "tacacs-secret"

web:
  port: 8080
  secret_key: "<auto-generated>"

api:
  port: 8081

portal:
  port: 8082
  session_duration_hours: 8
  wifi_ssid: "GuestWiFi"
  wifi_password: "guest1234"

profiler:
  enabled: true
  interface: "eth0"
  oui_db_path: "/var/lib/raspise/oui.csv"

display:
  enabled: true
  driver: "ili9341"        # ili9341 | st7789 | simulation
  rotation: 270            # degrees: 0 | 90 | 180 | 270
  brightness: 100          # 0-100 %
  cycle_interval: 8        # seconds between screens
  screens:
    - auth_log
    - stats
    - sessions
    - network
    - qr_code
```

## FreeRADIUS Integration (EAP-PEAP / EAP-TLS)

```bash
sudo bash scripts/setup_freeradius.sh
```

The script:
1. Installs FreeRADIUS with EAP + MSCHAPv2 modules
2. Configures `rlm_rest` to call RaspISE REST callbacks at `/radius/*`
3. Generates a self-signed TLS certificate for EAP
4. Imports RADIUS clients from your `config.yaml`
5. Disables the built-in RADIUS server (FreeRADIUS takes port 1812)

After setup, EAP-PEAP authentication flow:
```
Client → [EAP-PEAP] → FreeRADIUS → [REST /radius/auth] → RaspISE Policy Engine
                                                               │
                                                      VLAN assignment + log
                                                               │
FreeRADIUS ← Accept/Reject + Tunnel-Private-Group-Id ─────────┘
```

## Troubleshooting

| Symptom | Fix |
|---|---|
| Display blank after boot | Run `setup_display.sh` and reboot; check SPI with `ls /dev/spidev*` |
| RADIUS auth rejected with no log entry | Check NAS shared secret matches `radius.clients` in config |
| CHAP always fails | CHAP requires cleartext passwords — use PAP or EAP-PEAP instead |
| EAP-PEAP fails | Run `setup_freeradius.sh`; check `journalctl -u freeradius -f` |
| Guest portal not redirecting | Check iptables rule: `sudo iptables -t nat -L PREROUTING` |
| Service won't start (port in use) | Check for conflicting services: `ss -tulnp \| grep 1812` |
| High CPU on Pi Zero 2W | Set `profiler.enabled: false` if Scapy is overloading the CPU |
